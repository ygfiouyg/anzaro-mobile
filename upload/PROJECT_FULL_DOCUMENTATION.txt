1. نظرة عامة على المشروع
================================================================================

الاسم: Anzaro AI (أنزارو آي)
النوع: منصة ذكاء اصطناعي عربية شاملة
الـ Stack: Next.js 16 + TypeScript + Bun + Tailwind CSS 4 + shadcn/ui
الاستضافة: HuggingFace Spaces (Docker, cpu-basic, 16GB RAM, 50GB disk)
الـ Database: Prisma ORM + PostgreSQL/SQLite

================================================================================
2. البنية المعمارية (Architecture)
================================================================================

[المستخدم] → [HF Spaces (Next.js production)] → [AI Models (متعددة)] + [MCP Tools (355)]

الطبقات:
1. Frontend: Next.js App Router (React 19, Tailwind CSS 4, shadcn/ui)
2. Backend: Next.js API Routes (App Router)
3. AI Layer: موديلات متعددة (GLM, HuggingFace, GitHub, Groq, Gemini, etc.)
4. Tool Layer: MCP Registry (355 أداة) + Dynamic Tools (from GitHub)
5. Auth: NextAuth.js (33 OAuth providers) + Legacy Auth (Bearer token)
6. Database: Prisma ORM

8. نظام الـ Tool Calling
================================================================================

طريقة العمل (LLM-driven، NO keyword matching، NO ZAI fallback):

1. المستخدم يبعت رسالة
2. Pre-scan layer: يفحص لو فيه فعل أمر واضح
   - لو مفيش فعل أمر → skip tools (رد عادي)
   - لو فيه فعل أمر → execute الأداة المناسبة
3. Tool-calling layer: الـ LLM نفسه يقرر
   - بيبعتtools list + messages للـ API
   - الـ LLM بيرجع tool_calls أو content
   - لو tool_calls → execute → feed result → loop
4. Formatting: الـ LLM بيـ format النتيجة للمستخدم

ملفات رئيسية:
- src/lib/chat/chat-tool-calling.ts — الـ core layer
- src/app/api/chat/stream/route.ts — الـ integration

مهم: مفيش أي fallback لـ ZAI. كل موديل بيستخدم API بتاعه مباشرة.

9. نظام PDF Generation
================================================================================

Engine: Playwright (Chromium) — NOT pdf-lib
Pipeline: smart-doc-v2.ts → rendering-pipeline.ts → playwright-renderer.ts

Flow:
1. AI يولّد HTML + CSS
2. Playwright يفتح Chromium
3. renderHTMLToPDF() بيحوّل HTML → PDF
4. النتيجة تتسجل في /app/download/

Mutex: queue واحد بس (مش أكتر من Playwright instance واحد)

12. الـ Build Pipeline
================================================================================

Dockerfile steps:
1. FROM oven/bun:1
2. apt-get install (Playwright deps + Python + fonts)
3. pip3 install edge-tts
4. COPY package.json bun.lock
5. bun install --frozen-lockfile
6. COPY prisma
7. prisma generate
8. COPY . .
9. next build (production)
10. playwright install chromium
11. Create non-root user

start.sh:
- export NODE_ENV=production
- exec bunx next start -p 3000 -H 0.0.0.0

.dockerignore:
- node_modules, .git, tools/, skills/design/design-templates/
- Build context: 24MB (كان 80MB)

13. المشاكل المعروفة + الحلول
================================================================================

1. الـ chat بيرد "لم أتمكن":
   السبب: ZAI_API_KEY مش متاح
   الحل: أضفه في HF Spaces Settings

2. الـ sessions بتمسح كل restart:
   السبب: NEXTAUTH_SECRET مش متاح
   الحل: أضف NEXTAUTH_SECRET ثابت

3. الـ build بيـ crash في "Generating static pages":
   السبب: صفحات client-side بتـ crash في prerender
   الحل: force-dynamic على كل الصفحات

4. الـ tool-calling بيـ trigger على أسئلة عادية:
   السبب: keyword matching بدل intent detection
   الحل: الـ LLM نفسه بقرر (تم الإصلاح)

5. Google tools بتقول "غير مربوط":
   السبب: cookie decode بيـ fail
   الحل: readSessionFromCookie() + stable secret
