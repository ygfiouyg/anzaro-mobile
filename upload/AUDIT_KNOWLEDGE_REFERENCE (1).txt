3.1 CRITICAL BOTTLENECKS (P0 — fix immediately)

  [P0-1] chat/stream/route.ts — 3,235 lines, 9+ provider branches
    Impact: Slow streaming, hard to maintain, memory leaks in fallbacks
    Fix: Refactor into provider strategy pattern + streaming pipeline
    Target: Split into 5 files (providers/openai.ts, providers/hf.ts, etc.)

  [P0-2] Admin stats — 7 sequential db.message.count calls
    Impact: Admin dashboard loads in 3-5 seconds
    Fix: Replace with single db.message.groupBy
    Target: 1 query instead of 7 (700ms → 50ms)

  [P0-3] chat/conversations?messages=true — no pagination
    Impact: Loads ALL messages for 100 conversations (10MB+ response)
    Fix: Add cursor-based pagination (limit 50 messages per conversation)
    Target: Response size < 100KB

3.2 HIGH-PRIORITY ISSUES (P1)

  [P1-1] In-memory rate limiter doesn't aggregate across instances
    Impact: Rate limits bypassed on multi-instance deploy
    Fix: Wire Redis store (factory already exists in rate-limit.ts)

  [P1-2] In-memory session cache — 60s stale window on logout
    Impact: Logged-out users retain access for 60 seconds
    Fix: Use Redis for session cache + invalidate on logout

  [P1-3] Deferred DB writes in chat stream — lost on early disconnect
    Impact: User messages saved but AI responses lost
    Fix: Use AbortController + final flush on disconnect

  [P1-4] system-monitor.ts uses Set with O(n) linear scans
    Impact: High CPU under 1000+ connections
    Fix: Replace Set with Map for O(1) lookups

  [P1-5] Admin users — selects password hashes into memory
    Impact: Memory bloat + security risk
    Fix: Use select: { password: false } in Prisma query

3.3 MEDIUM-PRIORITY ISSUES (P2)

  [P2-1] AuthPage.tsx — 1,317 lines (too large)
    Fix: Split into LoginForm, RegisterForm, OtpForm, ResetPasswordForm

  [P2-2] ModelSelector.tsx — 2,177 lines
    Fix: Split into ModelCard, ModelGrid, ModelSearch, HfModelImporter

  [P2-3] chat-store.ts — 1,836 lines
    Fix: Split into conversationSlice, messageSlice, fileSlice

  [P2-4] html-template-generator.ts — 2,567 lines
    Fix: Split into template types + renderer

  [P2-5] No request body size cap on streaming routes
    Fix: Add 10MB body size limit in next.config.ts
